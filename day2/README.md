# Day 2: 参数控制与提示词工程

## 概述

第二天深入学习 OpenAI API 参数控制和提示词工程，学习如何精确控制模型输出。

## 学习目标

- 掌握 Temperature 参数对输出随机性的影响
- 理解提示词工程的基本原理
- 学会构建结构化的提示词
- 实现文本分类、信息提取等具体任务

## 核心概念

### 1. Temperature 参数
控制模型输出的随机性和创造性：

- **Low Temperature (0-0.3)**：输出更确定、一致
- **Medium Temperature (0.4-0.7)**：平衡创造性和确定性
- **High Temperature (0.8-1.0)**：输出更有创造性、多样化

### 2. 提示词工程
通过精心设计的提示词引导模型输出预期结果：

- **角色设定**：明确模型的身份和职责
- **任务描述**：清晰说明要完成的任务
- **示例引导**：提供期望的输入输出示例
- **输出格式**：指定输出的格式要求

### 3. 结构化输出
通过 JSON 格式或特定格式规范输出结果。

## 提示词模板

### 文本分类模板
```
你是一个专业的文本分类助手。

任务：将以下文本分类为以下类别之一：
- 正面情绪
- 负面情绪
- 中性情绪

文本：{text}

请只输出分类结果，不要其他解释。
```

```
你是一个信息提取专家。

- 人物姓名
- 日期
- 事件

文本：{text}

请以 JSON 格式输出：

```

## 参数调优指南

### 场景1：需要一致性输出
```python
temperature=0.0  # 最确定的结果
top_p=1.0
```

### 场景2：需要创造性输出
```python
temperature=0.8  # 更有创造性的结果
top_p=0.9
```

### 场景3：对话场景
```python
temperature=0.7  # 平衡创造性和一致性
```

## 最佳实践

### 1. 提示词设计原则
- ✅ 清晰明确的任务描述
- ✅ 提供具体的示例
- ✅ 指定输出格式
- ❌ 避免模糊不清的指令
- ❅ 避免过于复杂的提示词

### 2. 参数选择原则
- ✅ 根据任务类型选择合适的 temperature
- ✅ 使用 max_tokens 控制输出长度
- ✅ 测试多个参数组合
- ❌ 不要盲目使用高 temperature

### 3. 调试技巧
- 打印完整的请求和响应对
- 对比不同参数的结果
- 记录有效的提示词模板
